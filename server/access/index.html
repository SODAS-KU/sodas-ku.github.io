    Access

    In order to access the server, you need to have an account. Contact TBD with your basic information and SODAS affiliation, and they will walk you through the process.

    Connecting to the server

    You can connect to the server using SSH, either through the terminal or vscode.

    Terminal

    SSH

    ssh -L <local-port>:localhost:<remote-port> <username>@sodascomp01fl.unicph.domain

    To connect locally to the server on port 8000 with the user abc123, do:

    ssh -L 8000:localhost:8000 abc123@sodascomp01fl.unicph.domain

    Once you have a connection established, you can host a jupyter lab server on the server and connect to it from your local machine.

    Transferring files

    Transfer from local to server
    scp /path/to/file <username>@sodascomp01fl.unicph.domain:/path/to/destination

    To move a text file in documents to your H-drive which is mounted on the server by default, do:

    scp /documents/test.txt abc123@sodascomp01fl.unicph.doman:/ucph/hdir
    Transfer file from server to local
    scp <username>@sodascomp01fl.unicph.domain:/path/to/file /path/to/destination

    To move a text file from the H-drive on the server to your local documents folder, do:

    scp abc123@sodascomp01fl.unicph.doman:/ucph/hdir/test.txt /documents
